# 四类人群的组合场景

不继续堆新 Skill。下面的延伸全部由现有 Loop 组合，箭头表示上一步的验证产物成为下一步输入。

## 营销传播链路

1. 新品上市：`research-to-brief → product-gtm → content-repurpose / media-production → campaign-ops → kpi-monitor-diagnose`。
2. UGC / 达人项目（品牌/代理商且有授权采购入口）：`research-to-brief → creator-strategy-router → creator-portfolio → campaign-ops → content-repurpose / media-production → kpi-monitor-diagnose`。
3. 活动复盘再启动：`kpi-monitor-diagnose → research-to-brief（补冲突证据）→ campaign-ops`。

## 自媒体内容创作者

1. 账号定位与选题实验：`creator-strategy-router → platform-positioning-benchmark → content-repurpose / media-production → kpi-monitor-diagnose → platform-positioning-benchmark`。

没有采购权限的个人创作者、AI IP、艺人团队或产品团队，不进入 `creator-portfolio`；它们需要的是平台位置、参考系和实验设计，而不是采购名单。

1. 长内容拆条：`media-production（转写与人工选段）→ content-repurpose（平台稿）→ kpi-monitor-diagnose`。
2. 周更栏目：`research-to-brief（事实台账）→ content-repurpose → media-production（可选）→ kpi-monitor-diagnose`。
3. 品牌合作：`product-gtm（只运行声明矩阵）→ content-repurpose → media-production → campaign-ops（只做排期与就绪包）`。

## 软件 / 硬件产品 GTM

1. 定位实验：`research-to-brief → product-gtm → content-repurpose → kpi-monitor-diagnose → product-gtm`。
2. 版本发布：`office-action-desk（需求与风险）→ product-gtm → campaign-ops → content/media → KPI`。
3. 销售 / 客服证据包：`research-to-brief → product-gtm（claim matrix）→ content-repurpose（FAQ）→ office-action-desk（沟通草稿）`。

## 办公者

1. 会议邮件周报：`office-action-desk → research-to-brief（争议事实）→ office-action-desk（核实后的草稿）`。
2. 决策备忘：`kpi-monitor-diagnose → research-to-brief → office-action-desk`。
3. 跨团队上线：`office-action-desk（行动与风险）→ campaign-ops（工作包与就绪门）→ kpi-monitor-diagnose`。

## 跨 Loop 交接规则

- 交接的是 artifact manifest、当前 input generation、验证证据和未决异常，不是聊天摘要。
- 上游输入变更后，下游不得继续使用旧批准。
- 某个子 Loop 未完成时，编排器只能写“缺口”，不能把计划当完成。
- 所有外部动作仍停在 plan-only manifest。
