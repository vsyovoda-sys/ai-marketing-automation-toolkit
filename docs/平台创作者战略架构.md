# 平台创作者战略：入口、路径与证据边界

这套课程工具不把“找博主”和“找到自己/产品在平台上的位置”混为一件事。

```text
用户、目标、平台权限
          ↓
creator-strategy-router
     ┌────┴─────┐
     ↓          ↓
品牌/代理商     个人创作者、AI IP、产品GTM、无采购权限者
     ↓          ↓
creator-portfolio    platform-positioning-benchmark
采购组合与预算         内容定位、参照系与小实验
```

## 入口判断

不要按“这个人是不是博主”判断；按他要作出的决策和可合法使用的数据判断。

| 情况 | 应进入 | 最终交付 |
|---|---|---|
| 任何主体要在预算内选择合作创作者，且已证实具备官方采购后台、授权导出/API 或受托采购权限 | `creator-portfolio` | 合规候选快照、质量审计、评分契约、预算组合、选择理由、行动预览 |
| 个人创作者、AI IP、艺人/MCN、个人产品、创作者侧账号或无采购授权的产品GTM要找到内容位置与增长假设 | `platform-positioning-benchmark` | 平台机制证据账本、内容参照系、定位选项、内容实验与复盘口径 |
| 两者都需要 | 默认先定位；采购授权已证实时可并行 | 定位资料与采购候选池严格隔离，不能互相代替 |
| 没有采购权限且没有可用数据 | 路由器只交付缺口与可行来源 | 不生成假名单，不要求用户手工凑一张博主表 |

## 候选池由谁获得

对采购路径，Agent 的职责是先判断数据获取路径，而不是默认把“候选 CSV”这个麻烦交给用户：

1. 优先复用用户已有的合规导出。
2. 若用户已登录且具有官方采购页面、导出或 API 权限，当前 v1.1 只由 Agent 生成私有导出规范与字段边界；用户在授权系统中完成导出后，Agent 再验证本地导入结果并记录来源、时间、字段、用途和保留期。
3. 若不存在采购权限，停止采购路径，改做定位与内容参照系。

“页面能看见”不自动等于“可以批量抓取、长期保存或公开再分发”。课程不应默认把整页或整库账号资料发布到 GitHub。课程资料库应保存时间戳、来源链接、字段定义、聚合洞察与合规样例；受限账号数据只留在该用户的私有运行目录。真实平台连接器不随本版交付，未来必须另做平台条款、租户、凭据和速率限制审查。

## 平台算法研究如何进入 Loop

平台机制不是一个固定权重表。推荐系统会因用户、内容、场景、时间、审核和多样性策略而变化。

因此 `platform-positioning-benchmark` 要建立四栏证据账本：

| 证据等级 | 含义 | 可以做什么 |
|---|---|---|
| 官方 | 平台或监管公开的机制说明 | 作为研究边界与指标候选 |
| 可观察 | 用户自己账号或合规后台可见的数据 | 作为本账号实验的测量指标 |
| 假设 | 结合场景形成、尚待验证的判断 | 设计小实验 |
| 未知 | 没有公开或可验证证据 | 明确保留未知，不能编成权重 |

例如，CPE 是采购效率信号；它不等于小红书、抖音或任何平台的自然推荐权重。有关小红书、抖音、B站、微博、视频号和公众号的公开证据与不可声称事项，先见 [平台机制证据阅读说明](平台机制证据阅读说明.md)，再看 `skills/platform-positioning-benchmark/references/platform-mechanism-evidence.zh-CN.md` 的原始 URL。

## 课程里的正确使用方式

课程学员不必先学会抓取、采购后台或平台接口。先用 `templates/access-profile.example.json` 完成人工权限确认，再让 AI 运行 `creator-strategy-router`，回答主体、决策、平台和账号能力。AI 再交付正确的下一条 Loop 的最小输入清单。

这保证了：品牌采购者得到“选谁、花多少钱、为什么”；个人和产品得到“我在平台上该占据什么内容位置、先验证什么”。两条路径可以共用内容生态参照系，但不能混用数据权利和最终交付物。
