## 落地后的关键约束

1. 状态机记录输入指纹、尝试、产物、验证与允许的下一步，不依赖聊天记忆。
2. 重试有预算，第二次必须改变策略；否则进入异常队列。
3. 高风险审批绑定当前输入，不允许用旧批准覆盖新内容。
4. 评测集来自真实失败，但使用合成或匿名化最小样例。
5. 可观测日志只保存匿名 ID、哈希、错误类别和验证状态，不复制私有原文。
6. 解析器输出也要登记页数/表格/跳过项并抽检；“能读”不等于“读对”。
7. 秘密扫描不回显命中值，公开前还要检查 Git 历史。
8. 可清理不等于可删除；原文件默认保留。

## 未直接采用

- 不把庞大 agent 框架设为运行前提。
- 不封装 Cookie 复用、验证码规避、无上限抓取或自动评论。
- 不把模型自评分当验收。
- 不自动上传私有 trace 到第三方平台。
